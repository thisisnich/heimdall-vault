### Embedded System Definition

- An **embedded system** is a **processor-based electronic system** comprising both **hardware** and **software** designed to **perform specific tasks** with **limited resources**.
- Nearly any electronic system other than a desktop or laptop is an embedded system.
- Common examples: ATM machine, memory stick, keyboard, mouse, card reader, vending machine, etc.

**Key characteristics:**
- A processor-based electronic system ✓
- Comprises both hardware and software ✓
- Performs specific tasks with limited resources ✓

---

### The Need of Embedded System

Embedded systems are needed to achieve:

- **Increase Reliability**
- **Low Power**
- **Reduce Cost**
- **Shorter Response Time**
- **Smaller Form Factor**

---

### Processor and Peripherals

#### Processor

- The most important component responsible for **processing data** and **instructions**.
- In an embedded system context, a processor is usually in the **form of a chip**.
- Types:
  - **Microprocessor (MPU)**
  - **Microcontroller (MCU)**

#### Peripherals

- **Auxiliary electronic circuits** used by the processor for **communication purposes**.
- Can be in the form of a chip or **embedded within a chip**.
- Categories:
  - Digital Output and Input
  - Analog Output and Input
  - Clock and Timers
  - Serial Communications Interfaces

---

#### Digital Output and Input

- Also known as **General Purpose Input / Output (GPIO)** peripheral.
- **Digital output**: controls LEDs, light bulbs, or digital inputs of other devices.
- **Digital input**: reads the state of switches, push buttons, or digital status of other devices.

*Examples: RGB LED, Push Button*

---

#### Analog Output and Input

Common peripherals:
- **Pulse Width Modulator (PWM)** – generates audible tones through a buzzer.
- **Analog To Digital Converter (ADC)** – samples sensor data and converts it to a digital signal for processing.
- **Comparator** – measures and compares sensor data against a threshold value.

*Examples: Buzzer, Light Sensor*

---

#### Clock and Timers

Common peripherals:
- **Real Time Clock (RTC)** – provides accurate clock timing.
- **General Purpose Timer** – generates timer or counter events for triggering external systems or generating waveforms.

*Examples: Real Time Clock display, Countdown Timer*

---

#### Serial Communications Interfaces

Common peripherals:
- **UART (Universal Asynchronous Receiver / Transmitter)** – interfaces with another embedded system or a host computer.
- **SPI (Serial Peripheral Interface)** – typically used to interface with external memory IC chips.
- **I²C (Inter-Integrated Circuit)** – commonly used for interfacing with sensor IC chips with digital interfaces.

## 1.2 Embedded System Development Tools and Setup

### Development Tools

Development requires both **hardware** and **software** tools.

#### Hardware Tools

| Tool | Description |
|------|-------------|
| **Host PC** | A desktop or laptop used to install development software. |
| **Development Board** | An electronic board with a sample processor chip and peripheral connectors, enabling quick development without building the final PCB. |
| **Debug Probe** | A device connected between the host computer and the target processor/development board for probing, tracing, and controlling the target processor. |

#### Software Tools

All contained within an **Integrated Development Environment (IDE)**:

| Tool | Description |
|------|-------------|
| **Assembler** | Converts assembly/machine language into a pattern of bits the processor can execute. |
| **Compiler** | Translates high-level language (C/C++) into a lower-level language (assembly or machine language). |
| **Linker** | Links one or more object files generated by a compiler into a single executable, library, or object file. |
| **Debugger** | Tests and debugs software running on the target processor. |

---

### Development Setup

A typical embedded development setup connects:

```
Host PC (with IDE)
    |
    |--- USB Cable for Debugging --------> Debug Probe --> Development Board
    |
    |--- USB Cable with Power Supply for Serial Communications
```

## Embedded System Programming I

### C for Embedded System

- C is perhaps the **most popular** programming language in embedded systems.
- Advantages: **portability**, **better code efficiency with reduced overhead**, **shorter development time**.
- Offers **low-level hardware control** and is more readable than assembly language.
- Many C compilers are available for a wide variety of development platforms.

---

### C Program Structure and Sections

```c
// This is a simple program          ← Documentation Section

#include <stdint.h>                  ← Link Section

#define PI 3.14                      ← Definition Section

uint8_t ui8Counter;                  ← Global Declaration Section
uint32_t Function1 (uint8_t x);

int main (void)                      ← Main Function Section
{
    while(1);
    return 0;
}

uint32_t Function1 (uint8_t x)       ← Sub Program Section
{
    return result;
}
```

| Section | Purpose |
|---------|---------|
| Documentation Section | Comments describing the program |
| Link Section | `#include` header file inclusions |
| Definition Section | `#define` macro definitions |
| Global Declaration Section | Global variables and function prototypes |
| Main Function Section | The `main()` entry point |
| Sub Program Section | User-defined function definitions |

### Basic Data Type and Modifier

All data in C has an associated **type** describing its **interpretation**, **storage size**, and **range**.

**Basic types:** `char`, `int`, `float`, `double`

**Modifiers:** `signed`, `unsigned`, `short`, `long`

#### char Types

| Type | Storage | Value Range |
|------|---------|-------------|
| `char` | 8 bits | 0 to 255 or -128 to 127 |
| `unsigned char` | 8 bits | 0 to 255 |
| `signed char` | 8 bits | -128 to 127 |

#### Integer Types

| Type | Storage | Value Range |
|------|---------|-------------|
| `int` | at least 16 bits | -32768 to 32767 |
| `unsigned int` | at least 16 bits | 0 to 65535 |
| `signed int` | at least 16 bits | -32768 to 32767 |
| `short int` | at least 16 bits | -32768 to 32767 |
| `unsigned short int` | at least 16 bits | 0 to 65535 |
| `signed short int` | at least 16 bits | -32768 to 32767 |
| `long int` | at least 32 bits | -2147483648 to 2147483647 |
| `unsigned long int` | at least 32 bits | 0 to 4294967295 |
| `signed long int` | at least 32 bits | -2147483648 to 2147483647 |

#### Float and Double Types

| Type | Storage | Value Range | Precision |
|------|---------|-------------|-----------|
| `float` | 32 bits | ±1.2×10⁻³⁸ to ±3.4×10⁺³⁸ | 6 decimal places |
| `double` | 64 bits | ±2.2×10⁻³⁰⁸ to ±1.8×10⁺³⁰⁸ | 15 decimal places |
| `long double` | 80 bits | ±3.4×10⁻⁴⁹³² to ±1.1×10⁺⁴⁹³² | 19 decimal places |

**Example:** Best data type for the value `128` → **`unsigned char`** (range: 0–255)

### Variable, Array and Pointer

#### Variable

A **name** used to **refer to a location** in memory. Must be declared with a data type before use.

```c
unsigned char ucNumOfStudents;   // Variable Declaration
ucNumOfStudents = 24;            // Variable Assignment
```

#### Array

A **collection of data** of the **same data type** stored in a **common variable**. Data is stored linearly and referenced by an index.

```c
unsigned char ucStudentAge[5];   // Array Declaration (5 elements)
ucStudentAge[0] = 10;            // Array Assignment
```

#### Pointer

A **variable** that holds a **memory address** rather than data. Used to access data through memory addresses.

```c
unsigned char ucNumber;
unsigned char* ptrNumber;        // Pointer Declaration (using *)
ptrNumber = &ucNumber;           // Pointer Assignment (using &)
```

**Memory layout example:**

| Address | Content | Data Type | Variable | Value |
|---------|---------|-----------|----------|-------|
| 0xFFF0 | 0x41 | `char` | `cName` | `cName = 0x41` |
| 0xFFF1 | 0x10 | `char` | `cAge[3]` | `cAge[0] = 0x10` |
| 0xFFF2 | 0x11 | | | `cAge[1] = 0x11` |
| 0xFFF3 | 0x12 | | | `cAge[2] = 0x12` |
| 0xFFF4 | 0xFF | `char*` | `ptrName` | `ptrName = 0xFFF0` |
| 0xFFF5 | 0xF0 | | | |

**Example:** Declaration of an array of 100 unsigned integers named `uiBicycle`:
```c
unsigned int uiBicycle[100];
```

### Operators

#### Arithmetic Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `+` | Adds two operands | `a + b` |
| `-` | Subtracts second from first | `a - b` |
| `*` | Multiplies both operands | `a * b` |
| `/` | Divides and returns quotient | `a / b` |
| `%` | Divides and returns remainder | `a % b` |
| `++` | Increases integer value by one | `a++` |
| `--` | Decreases integer value by one | `b--` |

#### Comparison Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `==` | True if operands are equal | `a == b` |
| `!=` | True if operands are NOT equal | `a != b` |
| `>` | True if left > right | `a > b` |
| `<` | True if right > left | `a < b` |
| `>=` | True if left >= right | `a >= b` |
| `<=` | True if right >= left | `a <= b` |

#### Logical Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `&&` | AND: true if both operands are non-zero | `a && b` |
| `\|\|` | OR: true if any operand is non-zero | `a \|\| b` |
| `!` | NOT: reverses logical state | `!(a && b)` |

#### Bitwise Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `&` | Clear a bit when AND with 0 | `a & b` |
| `\|` | Set a bit when OR with 1 | `a \| b` |
| `^` | Invert a bit when XOR with 1 | `a ^ b` |
| `~` | Invert all bits | `~a` |
| `<<` | Left shift by N bits | `a << 5` |
| `>>` | Right shift by N bits | `b >> 5` |

#### Assignment Operators

| Operator | Description | Example |
|----------|-------------|---------|
| `=` | Assign right to left | `c = a + b` |
| `+=` | Add and assign | `c += a` |
| `-=` | Subtract and assign | `c -= b` |
| `*=` | Multiply and assign | `c *= a` |
| `/=` | Divide and assign quotient | `c /= b` |
| `%=` | Divide and assign remainder | `c %= a` |
| `<<=` | Left shift and assign | `c <<= 5` |
| `>>=` | Right shift and assign | `c >>= 5` |
| `&=` | Bitwise AND and assign | `c &= 5` |
| `\|=` | Bitwise OR and assign | `c \|= 5` |
| `^=` | Bitwise XOR and assign | `c ^= 5` |

**Example:**
```c
unsigned char ucNum1, ucNum2;
ucNum1 = 10 / 3;   // ucNum1 = 3
ucNum2 = 10 % 3;   // ucNum2 = 1
```

### Conditional Statements

Used to **decide the action of execution** of statements **based on certain conditions**.

Three types: `if`, `if-else`, `switch`

#### if Statement

```c
if (condition)
{
    Statement1;
}
Statement2;
```

If `condition` is true → execute `Statement1`, then `Statement2`. Otherwise, skip to `Statement2`.

#### if-else Statement

```c
if (condition)
{
    Statement1;
}
else
{
    Statement2;
}
Statement3;
```

Shorthand (ternary): `condition ? Statement1 : Statement2;`

#### switch Statement

```c
switch (variable)
{
    case value:
        Statement1;
        break;
    case value:
        Statement2;
        break;
    default:
        Statement3;
}
```

**Example:**
```c
unsigned char ucNum1, ucNum2;
ucNum1 = 10 * 3;          // ucNum1 = 30
if (ucNum1 == 30)
{
    ucNum2 = 10 - 3;      // ucNum2 = 7
}
ucNum1++;                 // ucNum1 = 31
// Result: ucNum1 = 31, ucNum2 = 7
```

## Embedded System Programming II

### Loop Statements

A loop allows **one or more statements to be executed multiple times**, reducing code length and saving memory.

Three types: `while`, `do-while`, `for`

#### while Loop

```c
while (condition)
{
    Statement1;
}
```

Condition is checked **before** each iteration.

#### do-while Loop

```c
do
{
    Statement1;
}
while (condition);
```

Statement executes **at least once**; condition is checked **after** each iteration.

#### for Loop

```c
for (init; condition; increment/decrement)
{
    Statement1;
}
```

**Example:**
```c
unsigned char ucNum1 = 0;
while (ucNum1 != 5)
{
    ucNum1++;
}
// Result: ucNum1 = 5
```

---

### Functions

A **function** is a block of code that performs a specific task. Every C program has at least one function: `main()`.

**Purpose:** Divide large code into smaller modules to improve **reusability** and **readability**.

**Two types:**
- **Standard C library functions** (e.g., from `stdlib.h`, `stdio.h`, `stdint.h`, `float.h`, `string.h`, `time.h`, `math.h`)
- **User-defined functions**

#### Function Structure

```c
// Function Declaration / Prototype
unsigned int square(unsigned int ucNum1);

// Function Definition
unsigned int square(unsigned int ucNum1)
{
    ...;
    return uiResult;
}

// Main Function with Function Call
int main(void)
{
    unsigned int uiAnswer = 0;
    uiAnswer = square(2);
    return 0;
}
```

#### Function Formats

- With arguments and with return value
- With arguments and without return value
- Without arguments and with return value
- Without arguments and without return value

---

#### Passing Arguments

**Pass by Value**
- A **copy of the data** is made and processed in the function.
- Changes do **not affect** the original data.

```c
unsigned int cube(unsigned int uiData)
{
    uiData = uiData * uiData * uiData;
    return uiData;
}
// uiNum remains 5; uiAnswer becomes 125
```

**Pass by Reference**
- The **address of the data** is passed to the function.
- Changes **do affect** the original data.

```c
void cube(unsigned int* ptrData)
{
    *ptrData = (*ptrData) * (*ptrData) * (*ptrData);
}
// uiNum becomes 125
```
