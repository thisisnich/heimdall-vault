## 📚 Chapter 2.1: Purpose of Project Management

### Card 1: Project Definition
**Q:** What is a project?

**A:** A unique effort with:
- Defined beginning and end
- Specific deliverables
- Defined resources

---

### Card 2: Project vs Business Process (Differences)
**Q:** What are the 3 key differences between a Project and a Business Process?

**A:**
| Project | Business Process |
|---------|------------------|
| Has a beginning and an end | Repeated over and over |
| Produces different output/deliverable | Produces same output each time |
| No regular work assignments | Has regular work assignments |

**Example:** Custom house (project) vs Standard house (business process)

---

### Card 3: Triple Constraint
**Q:** What are the 3 constraints in project management?

**A:**
1. **Time** (schedule/duration)
2. **Cost** (budget)
3. **Scope** (features/deliverables)

These must be balanced to maintain quality.

---

### Card 4: Duration Decreased ⚠️ QUIZ ANSWER
**Q:** What happens when the duration of the project is **decreased**?

**A:** 
- **Increase the cost** (hire more people/resources), AND/OR
- **Decrease the scope** (reduce features)

---

### Card 5: Cost Decreased ⚠️ QUIZ ANSWER
**Q:** What happens if the budget (cost) of the project is **decreased**?

**A:**
- **Increase the project duration** (reduce manpower), AND/OR
- **Decrease the scope** (reduce features)

---

### Card 6: Scope Increased ⚠️ QUIZ ANSWER
**Q:** What happens if project scope is **increased**?

**A:**
- **Increase project cost** (more resources needed), AND/OR
- **Increase project duration** (more time needed)

---

### Card 7: Project Management Goals
**Q:** What are the 3 main goals of project management?

**A:**
1. Complete project **on time**
2. Finish **within budget**
3. Achieve **customer satisfaction**

---

### Card 8: Project Phases
**Q:** What are the 5 phases in the full cycle of a project?

**A:**
1. Project initiation and organising
2. Project planning
3. Project execution
4. Project control and monitoring
5. Project closure

---

## 📚 Chapter 2.2: Four Phases of Project Management

### Card 9: Four Phases Overview
**Q:** What are the 4 main phases of project management (simplified)?

**A:**
1. **Initiation & Organisation**
2. **Planning**
3. **Execution, Monitoring & Control**
4. **Closure**

---

### Card 10: Phase 1 - Initiation Activities
**Q:** What are the key activities in Project Initiation & Organisation?

**A:**
- Organise a project team
- Define parameters (objectives, scope)
- Establish framework (how team operates)

---

### Card 11: Project Team - 3 Steps
**Q:** What are the 3 steps for organising a project team?

**A:**
1. Identify a **sponsor**
2. Appoint a **project manager (PM)**
3. Identify **core team** (max 7 members)

---

### Card 12: Sponsor Role
**Q:** What are the key responsibilities of a project sponsor?

**A:**
- Appoint project manager
- Set strategic direction
- Approve plan and budget
- Support team members
- Approve project changes
- Set priorities and resolve conflicts

---

### Card 13: Project Manager (PM) Role
**Q:** What are the key responsibilities of a project manager?

**A:**
- Define project management process
- Prepare and obtain approval of project plan
- Ensure team understands responsibilities
- Expend resources according to plan
- Analyse actual vs planned performance
- Establish priorities
- Provide status reports
- Resolve conflicts

---

### Card 14: Core Team Size
**Q:** What is the maximum recommended size for a core project team?

**A:** **At most 7 members**

Core members make major decisions; extended members provide specific expertise (no limit).

---

### Card 15: Project Authorization Reasons
**Q:** Name 5 reasons why projects are authorized.

**A:**
1. **Technology advances** (e.g., efficient car for gas shortage)
2. **Social needs** (e.g., new substation for industrial park)
3. **Market demands** (e.g., new video game player)
4. **Business needs** (e.g., toxic material guidelines)
5. **Customer requests** (e.g., portable water project)

---

### Card 16: SWOT Analysis
**Q:** What does SWOT stand for and what is its purpose?

**A:**
- **S**trengths
- **W**eaknesses
- **O**pportunities
- **T**hreats

**Purpose:** Identify suitable projects by analyzing internal capabilities and external factors.

---

### Card 17: Project Objective Statement
**Q:** What should a Project Objective Statement include?

**A:**
- **What** you're going to do (scope)
- **By when** (schedule)
- **With how many** (resources/costs)

**Max 25 words, only 1 target, clear and precise**

Example: "Implement integrated employee database by 31 Dec 2006 with budget of S$20 million."

---

### Card 18: Flexibility Matrix
**Q:** What are the 3 flexibility levels in a Flexibility Matrix?

**A:**
1. **Least Flexible** — cannot be exceeded
2. **Moderately Flexible** — can be adjusted within project objective
3. **Most Flexible** — management willing to exceed original goal if necessary

Used to determine trade-offs among scope, schedule, and resources.

---

### Card 19: Major Deliverables (MD)
**Q:** What is the purpose of a List of Major Deliverables?

**A:**
- Defines what the project **is expected to produce**
- Provides scope and boundaries
- Reduces rework cycle

---

### Card 20: Project Charter
**Q:** What is a Project Charter and what does it contain?

**A:**
**Definition:** A document that formally authorizes a project and gives PM authority.

**Contains:**
- Project title and description
- PM assigned and authority level
- Project objective statement
- Business case (why?)
- Product description/deliverables

---

### Card 21: Project Framework - 4 Areas
**Q:** What 4 areas should a project framework encompass?

**A:**
1. **Planning** (what tools, who develops plan)
2. **Tracking** (meetings, progress assessment)
3. **Practices** (project file, change approval)
4. **Relationships** (communication, conflict resolution)

---

### Card 22: Phase 2 - Planning Activities
**Q:** What are the key activities in Project Planning?

**A:**
1. Develop **Work Breakdown Structure (WBS)**
2. Approximate **duration of tasks**
3. Generate **Gantt Chart**

---

### Card 23: WBS Definition
**Q:** What is a Work Breakdown Structure (WBS)?

**A:** A tool for identifying and presenting project work in a **hierarchical and logically structured manner**.

**Purpose:** Ensure all work required to meet objectives is properly identified, defined, and assigned.

---

### Card 24: WBS Key Elements
**Q:** What are the 4 key elements of WBS?

**A:**
1. Sequence is **NOT** important
2. Completeness **IS** important
3. Verb/noun symbolizes task & activity
4. Each breakdown needs minimum of **2 tasks**

---

### Card 25: WBS Development Approach
**Q:** What approach is used to develop a WBS?

**A:** **Top-down approach:**
1. Identify 4 to 7 major components
2. Group by: product deliverables, life cycle phases, functional responsibilities, or geographical location
3. Validate using **bottom-up approach**
4. WBS is a **team process** (not PM alone)

---

### Card 26: Task Duration Assignment
**Q:** Who assigns the duration of tasks in a project?

**A:** Durations are assigned by the **owners** of the tasks (not the PM).

**Key principles:**
- Avoid padding — use most likely duration
- Document all assumptions
- Document number of persons and training requirements

---

### Card 27: Effort → Duration → Calendar Time
**Q:** Explain the relationship: Effort → Duration → Calendar Time

**A:**
- **Effort** — total work required (person-hours); for charging purposes
- **Duration** — time to complete task; for schedule purposes (considers productivity & wait time)
- **Calendar Time** — real elapsed time; for tracking (considers non-working days, weekends, holidays)

**Formula:** Effort ÷ Resources = Duration × Resource Availability = Calendar Time

---

### Card 28: Gantt Chart
**Q:** What is a Gantt Chart and what is its purpose?

**A:**
**Definition:** A simple representation of a project showing time taken and resources required for each activity.

**Purpose:**
- Useful **communication tool**
- Easy to understand
- Provides overview of responsibility and progress

---

### Card 29: Phase 3 - Execution & Monitoring
**Q:** What are the 4 main activities in Project Execution, Monitoring & Control?

**A:**
1. **Monitor** the environment (collect info & update status)
2. **Manage** change (analyze variance)
3. **Track** progress (take adaptive action)
4. **Communicate** progress (report status)

---

### Card 30: Variance Definition
**Q:** What is variance in project management?

**A:** When actual activities **deviate from planned activities**.

After identifying causes and impacts, the PM must plan for **adaptive action**.

---

### Card 31: Adaptive Action Planning
**Q:** What should you do when planning for adaptive action?

**A:**
- Involve team members in planning
- Review project change control process
- Re-visit all assumptions (schedule, resource, scope)
- Check status data for accuracy
- Implement contingency plan
- Hold key milestones while re-arranging tasks
- Re-plan/re-estimate schedule or resources

---

### Card 32: Adaptive Action Implementation
**Q:** What should you do when taking adaptive action?

**A:**
- Implement new plan
- Communicate changes to team
- Get buy-in from project team
- Document all changes

**Remember:** Changes in one aspect (scope/schedule/resource) **will affect others** — monitor carefully!

---

### Card 33: Project Status Reporting
**Q:** How often should project status be reported and what should be included?

**A:**
**Frequency:** At least **once a month**

**Include:**
- Overall % of completion (emphasize milestones)
- Major tasks completed this period
- Project analysis, variance, and adaptive action
- Major tasks for next period
- Outstanding issues (updated schedules, milestones, costs)

---

### Card 34: Phase 4 - Project Closure
**Q:** What are the key considerations before project closure?

**A:**
- **Announce** end of project
- **Conduct** final project review
- **Debrief** key learning
- **Deliver** acknowledgements and rewards
- **Closing ritual** (celebration)
- **Complete paperwork** (contract, payment, schedule, lessons learned, close file)

---

### Card 35: Conduct Review Purpose
**Q:** What is the purpose of conducting a project review?

**A:**
- Highlight **what went right AND what went wrong**
- Purpose is **NOT finger pointing** but to minimize future problems
- Essential for **continuous improvement**
- Identify success elements
- Avoid future problems
- **Capture key learning** for future projects

---

### Card 36: Close Project File
**Q:** What steps are involved in closing the project file?

**A:**
- Complete all final review documents
- Bundle and label all status reports
- Send all project files for archiving
- **PM is responsible** for ensuring proper storage

## 📚 PERT and CPM

### Card 37: PERT Definition
**Q:** What does PERT stand for and what is its purpose?

**A:**
- **PERT = Program Evaluation and Review Technique**
- Used when task durations are **uncertain** or variable
- Uses **probabilistic time estimates** to calculate expected project duration
- Best for R&D, new product development, IoT prototyping

---

### Card 38: PERT Time Estimates
**Q:** What are the 3 time estimates used in PERT?

**A:**
1. **Optimistic (O)** — Best-case scenario (everything goes perfectly)
2. **Most Likely (M)** — Normal scenario (most probable duration)
3. **Pessimistic (P)** — Worst-case scenario (everything goes wrong)

---

### Card 39: PERT Expected Time Formula ⚠️ QUIZ ANSWER
**Q:** What is the formula for calculating Expected Time (TE) in PERT?

**A:** **TE = (O + 4M + P) / 6**

Where:
- O = Optimistic time
- M = Most Likely time
- P = Pessimistic time

---

### Card 40: PERT Variance Formula
**Q:** What are the formulas for Variance and Standard Deviation in PERT?

**A:**
- **Standard Deviation (σ) = (P - O) / 6**
- **Variance (σ²) = [(P - O) / 6]²**

---

### Card 41: PERT Calculation Example
**Q:** Given: Optimistic = 3 days, Most Likely = 5 days, Pessimistic = 13 days. Calculate TE and σ.

**A:**
- **TE = (3 + 4×5 + 13) / 6 = (3 + 20 + 13) / 6 = 36 / 6 = 6 days**
- **σ = (13 - 3) / 6 = 10 / 6 ≈ 1.67 days**
- **σ² ≈ 2.78**

---

### Card 42: CPM Definition
**Q:** What does CPM stand for and what is its purpose?

**A:**
- **CPM = Critical Path Method**
- Used when task durations are **deterministic** and well-known
- Identifies the **critical path** — the longest path that determines project duration
- Best for construction, manufacturing, well-defined projects

---

### Card 43: CPM Key Features
**Q:** What are the key features of CPM?

**A:**
- Uses **single time estimate** for each activity (deterministic)
- Identifies **critical activities** — activities with zero slack/float
- Calculates **ES, EF, LS, LF** (earliest/latest start/finish times)
- Determines **slack/float** — amount of time an activity can be delayed without delaying the project

---

### Card 44: CPM Forward Pass
**Q:** What is calculated in the Forward Pass of CPM?

**A:**
- **Earliest Start (ES)** and **Earliest Finish (EF)**
- Calculated **left to right** through the network diagram
- **ES = max(EF of predecessors)**
- **EF = ES + Duration**

---

### Card 45: CPM Backward Pass
**Q:** What is calculated in the Backward Pass of CPM?

**A:**
- **Latest Start (LS)** and **Latest Finish (LF)**
- Calculated **right to left** through the network diagram
- **LF = min(LS of successors)**
- **LS = LF - Duration**

---

### Card 46: Slack/Float Formula
**Q:** What is the formula for calculating Slack/Float in CPM?

**A:** **Slack = LS - ES** or **Slack = LF - EF**

Activities with **zero slack** are on the **critical path**.

---

### Card 47: Critical Path Definition
**Q:** What is the critical path in a project?

**A:**
- The **longest path** through the network diagram
- The sequence of tasks that determines the **minimum project duration**
- Activities on the critical path have **zero slack/float**
- If any activity on this path is delayed, the **entire project is delayed**

---

### Card 48: PERT vs CPM Comparison
**Q:** What are the key differences between PERT and CPM?

**A:**
| Aspect | PERT | CPM |
|---|---|---|
| Time Estimates | 3 estimates (O, M, P) | 1 estimate (deterministic) |
| Uncertainty | High uncertainty | Low uncertainty |
| Focus | Probability of completion | Critical path identification |
| Best For | R&D, new projects, IoT prototyping | Construction, manufacturing |
| Complexity | More complex | Simpler |

---

### Card 49: Network Diagram Dependencies
**Q:** What are the 4 types of dependencies in a network diagram?

**A:**
1. **Finish-to-Start (FS)** — Task A must finish before Task B starts (most common)
2. **Start-to-Start (SS)** — Task A must start before Task B can start
3. **Finish-to-Finish (FF)** — Task A must finish before Task B can finish
4. **Start-to-Finish (SF)** — Task A must start before Task B can finish (rare)

---

### Card 50: Slack/Float Definition
**Q:** What is slack/float in project management?

**A:** The amount of time an activity can be **delayed without delaying the project**.

Activities **not** on the critical path have slack. Activities on the critical path have **zero slack**.

---

### Card 51: IoT Project - When to Use PERT
**Q:** When should you use PERT for an IoT project?

**A:**
- Developing **new sensor technology** with unknown performance
- **Wireless protocol selection** with uncertain range/reliability
- **Cloud integration** with third-party services
- **Regulatory certification** with unpredictable timelines

---

### Card 52: IoT Project - When to Use CPM
**Q:** When should you use CPM for an IoT project?

**A:**
- **Hardware assembly** with known component lead times
- **Software testing** with established test suites
- **Documentation** with clear requirements
- **Packaging and shipping** with standard processes

---

### Card 53: Hybrid Approach
**Q:** What is the hybrid approach for IoT projects?

**A:** Many IoT projects use **PERT for R&D phases** and **CPM for production phases**:
- **Design/Prototype phase:** PERT (high uncertainty)
- **Development phase:** PERT for new tech, CPM for standard tasks
- **Testing phase:** CPM (well-defined test procedures)
- **Production/Deployment:** CPM (standardized processes)
