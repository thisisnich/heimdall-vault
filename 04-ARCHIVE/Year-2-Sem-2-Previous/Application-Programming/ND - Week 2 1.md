# Week 2: User Interface and Event-Driven Programming

##  User Interface (UI)

### UI Types
- **Text-based UI:** Command-line and terminal interfaces
- **Graphical UI (GUI):** Visual interfaces with graphical elements

### Modal Dialogs
- **Definition:** A modal is a dialog box that appears above all other windows
- **Characteristics:**
  - User must interact with it to continue working
  - Cannot select windows behind the modal
  - Requires user action before proceeding

### Advanced Input Methods
- **Camera:** Visual input and image recognition
- **Gestures:** Touch and motion-based interactions
- **Voice:** Speech recognition and voice commands

---

##  Event-Driven Programming

### Event-Driven Architecture
- **Definition:** Programming paradigm controlled by user or system actions
- **Process:** User actions are translated into events
- **Examples:**
  - Click events
  - Double-click events
  - Key down events
  - Mouse movement events

### Event Handling
- **Event Listeners:** Code that waits for specific events
- **Event Handlers:** Functions that execute when events occur
- **Event Loop:** Continuous monitoring for user interactions

---

##  Implementation Concepts

### GUI Development
- **Form Design:** Creating visual interfaces
- **Control Layout:** Positioning UI elements
- **Event Binding:** Connecting actions to UI elements

### User Experience
- **Responsive Design:** Adapting to different screen sizes
- **Accessibility:** Ensuring usability for all users
- **Intuitive Navigation:** Clear and logical interface flow

---

##  Key Takeaways

- **UI Types:** Understanding text-based vs graphical interfaces
- **Modal Dialogs:** Managing user attention and workflow
- **Event-Driven Programming:** Responding to user actions
- **Modern Input Methods:** Beyond traditional keyboard/mouse
- **User Experience:** Creating intuitive and accessible interfaces
