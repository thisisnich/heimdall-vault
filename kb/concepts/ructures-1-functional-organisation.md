## 2.2.1 (continued) — Define Project Parameters

The purpose of defining project parameters is to **ensure common understanding of the project and objectives among all team members**. Key documents include:

### Project Objective Statement

States **what** you are going to do (scope), **by when** (schedule), and **with how many** (resources/costs). Should be at most 25 words with only 1 target, written in clear and precise language.

> *Example: "Implement an integrated employee system database by 31 Dec 2006 with a budget of S$20 million."*

### Flexibility Matrix

Helps the team determine trade-offs among scope, schedule and resources:

| | Least Flexible | Moderately Flexible | Most Flexible |
|---|---|---|---|
| Schedule | | ✓ | |
| Scope | ✓ | | |
| Resource | | | ✓ |

- **Least flexible** — cannot be exceeded
- **Moderately flexible** — can be maximised or minimised within project objective
- **Most flexible** — management willing to exceed original goal if necessary
