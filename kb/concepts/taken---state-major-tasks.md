## 2.2.4 Project Closure

After the customer accepts the final deliverables, the PM must still go through the **process of project closure**.

Key considerations:
- **Announce** end of project
- **Conduct** final project review
- **Debrief** key learning
- **Deliver** acknowledgements and rewards
- **Closing ritual** (celebration)
- **Complete paperwork:**
  - Contract
  - Payment
  - Schedule update
  - Document lessons learnt
  - Close project file

### Conduct Review

Conduct a review that highlights both **what went right and what went wrong**. Purpose is **not finger pointing** but to minimise problems in future.

- Essential for **continuous improvements**
- Identify elements contributing to project success
- Avoid problems in future and identify processes needing improvement
- **Capture key learning** which are invaluable for future project planning

### Close the Project File

- Complete all final review documents
- Bundle all status reports and label them to ensure individual papers are not lost
- Send all project files for archiving
- PM is responsible for ensuring all project files are stored properly
