# Pical Considerations Include   Functions

- pical considerations include:
  - Functions
  - Software code size
  - Operating hours, MTBF
  - Dimensions, size, weight
  - Power requirements (e.g. 28VDC, 115VAC, 400Hz 3-phase)
  - Environmental factors (temperature, pressure, humidity, etc.)

### System Design Review (SDR)

- Follows immediately after the PSD phase
- A **top-level review of system hardware characteristics** so that preliminary design may proceed with confidence
- Key hardware characteristics are reviewed to ensure **no major mismatches** between system requirements and hardware capabilities

### Software Specification Review (SSR)

- Similar process to hardware SDR, but focuses on **software requirements analysis**
- Identifies any limitations such as throughput, timing or memory imposed by the hardware solution
- Both SDR and SSR allow the preliminary design to develop up to the **Preliminary Design Review (PDR)**

### Preliminary Design Review (PDR)

- **First detailed review** of the initial design (both hardware and software) versus system requirements
- Includes **preliminary tests and/or software simulation**
- Usually the **last review before committing major design resources** to the detailed design process and committing major program resources and investment

### Critical Design Review (CDR)

- By CDR, **major effort, resources, and investment** will have been committed to the program
- Identify **final design flaws**, or trade-off risks of one path implementation versus another
